# Asynchronous-FIFO-Verilog
Asynchronous FIFO design in Verilog with Gray code pointers and clock-domain crossing synchronization, verified using Icarus Verilog and GTKWave.
## Asynchronous FIFO — Theory & Working

This project implements an **Asynchronous FIFO (First-In First-Out)** buffer in Verilog that allows safe data transfer between two different clock domains.
The design uses independent write (`wr_clk`) and read (`rd_clk`) clocks, making it suitable for systems where producer and consumer logic operate at different 
speeds. A dual-port memory array is used to store data, allowing simultaneous read and write operations.

To safely transfer pointer information across clock domains, the FIFO uses **Gray code pointers** instead of binary counters. Gray coding ensures that only
one bit changes at a time, reducing metastability risks during clock-domain crossing (CDC). The write pointer is synchronized into the read clock domain,
and the read pointer is synchronized into the write clock domain using two-stage flip-flop synchronizers. This guarantees reliable detection of FIFO status 
without data corruption.

The FIFO depth is 16 entries, implemented using a 5-bit pointer where the extra MSB is used for wrap detection. The lower 4 bits address memory locations,
while the MSB tracks circular buffer overflow. The `empty` flag is asserted when the read pointer equals the synchronized write pointer, indicating no
available data. The `full` flag is generated by comparing the next write pointer with the read pointer while inverting the upper bits, allowing accurate
detection when the FIFO reaches maximum capacity.

During operation, data is written into memory when `wr_en` is high and the FIFO is not full. Similarly, data is read when `rd_en` is asserted and the FIFO 
is not empty. The testbench verifies functionality by running write and read clocks at different frequencies, writing sequential data into the FIFO, and
then reading it back while observing the `full`, `empty`, and `rd_data` signals in GTKWave.

This design demonstrates key digital design concepts such as clock-domain crossing (CDC), Gray code synchronization, circular buffer management, and safe
asynchronous communication between independent timing domains.
